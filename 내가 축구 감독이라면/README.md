# 내가 축구 감독이라면

2026 월간 해커톤 **「내가 축구 감독이라면 - 월드컵 전술 웹서비스 챌린지」** 출품 프로젝트입니다.

**배포 데모:** https://jinwon25.github.io/Dacon/

## 콘셉트: RE:TACTIC Match Intervention Lab

> 실제 월드컵 경기의 결정적 순간을 데이터로 진단하고, 포메이션·역할·압박·교체를 조정해 전술 개입안의 이점과 리스크를 비교하는 코칭 의사결정 워크스페이스

단순히 선수를 전술판에 올리는 데서 끝내지 않고 다음 흐름을 하나의 짧은 플레이 경험으로 연결합니다.

1. 실제 월드컵 경기와 도전 시나리오 선택
2. 상대의 공격 경로와 핵심 선수에 관한 데이터 브리핑 확인
3. 드래그 앤 드롭으로 선발 명단과 포메이션 구성
4. 경기 중 결정적 순간에 적용할 전술·선수 역할·교체 설계
5. 실제 관측값을 기준선으로 전술안의 침투·순환·노출·부담 비교
6. 이점, 리스크, 운영 조건이 담긴 `전술 개입 메모` 확인

## 핵심 가치

- **즉시 이해되는 재미:** 첫 화면에서 바로 유명 경기의 위기 상황에 도전합니다.
- **감독다운 선택:** 위치뿐 아니라 선수 역할, 압박 높이, 수비 폭, 공격 속도까지 조정합니다.
- **데이터의 의미 있는 사용:** 실제 경기 데이터가 장식용 차트가 아니라 전술 선택의 근거와 결과 계산에 쓰입니다.
- **현장에서 읽히는 결과:** 전술안의 채택 판단, 이점, 리스크, 운영 조건을 한 장의 메모로 요약합니다.

## 로컬 실행

Node.js 18 이상이 필요합니다.

```bash
npm install
npm run dev
```

프로덕션 빌드와 타입 검사는 다음 명령으로 확인합니다.

```bash
npm run typecheck
npm run build
npm run verify
```

## 구현된 플레이 흐름

### 실제 경기 분석

- 한국 대 포르투갈 경기 상황으로 바로 시작
- StatsBomb 실제 이벤트로 계산한 직전 20분 코치 브리핑
- 양 팀의 실제 공격 지역 진입 패스와 슈팅 위치 지도
- 4개 포메이션 프리셋과 선수 드래그 배치
- 압박, 공격 폭, 템포, 위험 감수 조절
- 선수 역할 변경 및 1회 교체
- 선택에 실시간 반응하는 전술 시나리오 비교 지표
- 채택 판단, 이점, 리스크, 운영 조건이 담긴 의사결정 메모
- StatsBomb 360의 실제 패스·슈팅 프리즈프레임과 제안 개입을 분리한 2D 전술 시퀀스

### 범용 전술 스튜디오

- 팀명, 상대, 경기 단계, 시간, 스코어, 전술 목표 입력
- 대한민국·아르헨티나·프랑스·코너킥 전술 템플릿
- 4개 포메이션 프리셋과 자유 드래그 배치
- 선수 표시 이름, 등번호, 역할 편집
- 벤치 선수 교체
- 상대 포메이션 오버레이와 상대 선수 자유 배치
- 패스·침투 화살표와 장면별 경로 저장
- 최대 5개 장면 타임라인과 순차 애니메이션 재생
- 저장 장면의 기존 위치를 점선 마커로 표시하는 Before/After 비교
- 팀 폭, 수비 라인, 전환 위험 실시간 구조 점검
- 선택 선수의 주변 패스 선택지·선수 간격·공격 지역 수적 관계 진단
- 규칙 기반 코치 제안 미리보기·원클릭 적용
- 전술안 A/B 저장·불러오기·차이 비교와 배치 실행 취소
- 브라우저 자동 저장, 초기화, PNG 및 인쇄·PDF 저장

상단 모드 스위치 또는 첫 화면의 `빈 전술판에서 시작`으로 두 모드를 오갈 수 있습니다.

검수 화면은 [`artifacts`](artifacts) 폴더에 보관합니다. `#briefing`, `#tactics`, `#result` 해시를 붙이면 개발 중 해당 화면으로 바로 진입할 수 있습니다.

## 기술 구성

- React 18 + TypeScript + Vite
- 외부 UI 프레임워크 없이 제작한 CSS·SVG 전술 시각화
- Pointer Events 기반 마우스·터치 드래그
- `localStorage` 기반 자동 저장
- `html-to-image` 기반 16:9 전술안 PNG 생성
- GitHub Actions + GitHub Pages 정적 배포

실행 중 외부 API나 API 키가 필요하지 않습니다. 실제 경기에서 검증한 소규모 이벤트·360 파생 데이터만 정적 번들에 포함합니다. 360 값은 이벤트 시점의 가시 선수 스냅샷이며 연속 트래킹 데이터로 표현하지 않습니다.
