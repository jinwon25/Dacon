# RE:TACTIC 프로젝트 전략 초안

## 1. 한 줄 소개

**“그때 내가 감독이었다면?” 실제 월드컵 경기의 결정적 순간을 데이터로 진단하고 직접 전술 개입안을 설계·비교하는 코칭 의사결정 워크스페이스.**

## 2. 해결하려는 경험

일반적인 전술 보드는 선수를 배치한 뒤 경험이 끝나기 쉽습니다. RE:TACTIC은 `상황 진단 → 전술 설계 → 리스크 비교 → 운영 조건 확인`의 루프를 만들어 사용자가 코칭 스태프처럼 근거를 바탕으로 개입안을 완성하게 합니다.

첫 방문자는 축구 지식이 많지 않아도 추천 지시와 간단한 설명을 통해 플레이할 수 있고, 축구 팬은 세부 전술 설정과 데이터 비교를 통해 더 깊게 즐길 수 있도록 설계합니다.

## 3. 대표 사용자 흐름

### Step 1. 오늘의 경기

- 실제 월드컵 경기 기반 도전 시나리오를 카드로 제시
- 각 카드에는 점수, 시간, 위기 상황, 도전 목표를 표시
- 예: “후반 60분, 한 골이 필요합니다. 공격 균형을 깨지 않고 동점을 만드세요.”

### Step 2. 데이터 브리핑

- 상대의 주요 공격 방향, 슈팅 위치, 핵심 연결 선수 표시
- 차트마다 한 문장의 해석을 함께 제공
- 데이터에 익숙하지 않은 사용자를 위해 `코치의 한마디` 제공

### Step 3. 전술 보드

- 선수를 피치 위로 드래그 앤 드롭
- 포메이션 프리셋 선택 후 자유 이동 가능
- 선수 클릭 시 역할과 개인 지시 설정
- 팀 단위 설정: 압박 높이, 수비 폭, 공격 속도, 패스 성향
- 잘못된 배치나 위험한 선택은 실시간 경고하되 선택 자체를 막지는 않음

### Step 4. 결정적 순간

- 2D 탑다운 경기 화면에서 짧은 시퀀스 재생
- 특정 시점에 경기가 멈추고 사용자가 교체 또는 전술 변경
- 선택 전후의 공간, 패스 경로, 수적 우위를 시각적으로 비교

### Step 5. 전술 개입 메모

- 전술안 채택 판단과 핵심 운영 조건
- 침투 가능성, 볼 순환 안정성, 전환 노출, 고강도 부담 비교
- 실제 경기 선택 또는 기준 전술과의 사후 비교
- 시나리오 점수는 승률이나 실제 경기 결과 예측이 아님을 명시

## 4. 심사 기준 대응

| 평가 항목 | 대응 설계 |
|---|---|
| 참신성 30 | 실제 경기의 결정적 순간에서 관측 데이터로 문제를 진단하고 개입안을 비교하는 워크스페이스 |
| 감독 경험 25 | 브리핑, 라인업, 터치라인 개입, 경기 후 리포트까지 하나의 서사로 연결 |
| 완성도 25 | 광범위한 매치 엔진 대신 검증된 소수 경기의 데이터·전술 비교를 정교하게 구현 |
| 기획/구현 20 | 모든 데이터와 인터랙션을 `전술 선택의 근거와 결과`라는 한 축으로 통일 |

## 5. MVP 범위

완성도를 위해 최초 버전은 다음으로 제한합니다.

- 대표 경기 시나리오 1개
- 두 국가대표팀과 교체 선수 포함 로스터
- 포메이션 프리셋 7개: 4-3-3, 4-2-3-1, 4-4-2, 4-1-4-1, 4-3-1-2, 3-4-3, 5-3-2
- 드래그 앤 드롭 배치
- 팀 전술 설정 4종
- 개인 역할 설정 2~3종
- 전술 선택에 따라 달라지는 4개 시나리오 비교 지표
- 전술 개입 메모와 인쇄용 요약 카드

두 번째 시나리오, 생성형 AI 코치, 자유 경기 생성은 핵심 MVP가 안정된 뒤 추가합니다.

## 6. 데이터 원칙

- 실제 선수 이름, 포지션, 국가, 경기 스코어와 공개된 경기 통계를 출처와 함께 사용
- 라이선스가 불명확한 능력치나 게임 데이터를 복제하지 않음
- 세부 전술 능력치는 공개 통계로부터 직접 산출하거나 `서비스용 추정 지표`임을 명시
- 외부 API에 런타임 의존하지 않고 검증한 데이터를 정적 JSON으로 포함
- 데이터 출처, 가공 방식, 업데이트 일자를 저장소와 서비스에 표시

## 7. 확정 기술 구성

- React + TypeScript + Vite
- 자체 CSS·SVG로 만든 반응형 전술판과 데이터 시각화
- Pointer Events 기반 마우스·터치 드래그
- React 로컬 상태와 `localStorage` 자동 저장
- `html-to-image` 기반 전술안 PNG 생성
- 정적 관측 데이터 + 투명한 휴리스틱 기반 시나리오 비교
- GitHub Actions + GitHub Pages 배포

핵심 비교 계산은 같은 입력에 항상 같은 결과가 나오도록 구성하고, 실제 관측값과 휴리스틱 결과를 구분해 표시합니다.

## 9. 가장 먼저 검증할 것

1. 사용자가 설명 없이도 선수를 옮기고 전술을 저장할 수 있는가?
2. 전술 변경 전후의 차이를 10초 안에 알아볼 수 있는가?
3. 결과가 무작위처럼 보이지 않고 데이터와 선택에서 비롯되었다고 느끼는가?
