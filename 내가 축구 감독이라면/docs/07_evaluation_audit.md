## 항목별 확인

### 1. 참신성 30점

- 단순 전술판이 아니라 실제 경기의 특정 시점에 개입하는 `Match Intervention Lab`이다.
- 한국–포르투갈은 결승골, 아르헨티나–네덜란드는 리드 보호, 2026 남아공–한국은 실점 직후 추격 설계로 의사결정 유형이 다르다.
- 실제 이벤트 좌표의 공격 진입·슈팅·패스 네트워크·볼 흐름이 전술 선택의 근거가 된다.
- 2026 경기는 FIFA 공식 전체 경기 보고서의 경기 국면과 상위 패스 연결을 사용하며 공개되지 않은 좌표를 만들지 않는다.
- 사용자의 대형·배치 높이와 폭·역할·교체·팀 지시를 실제 관측 기준선과 다시 비교한다.
- 결과는 임의 스코어가 아니라 채택 판단, 이점, 리스크, 운영 조건이 담긴 코칭 메모다.

### 2. 감독 경험 설계 25점

- 첫 화면에서 경기 시간, 점수, 목표를 이해하고 시나리오를 고른다.
- 브리핑에서는 숫자만 보여주지 않고 `무슨 문제인가`를 한 문장으로 해석한다.
- 포메이션은 많은 버튼 대신 설명이 붙은 단일 선택 컨트롤로 축약했다.
- 선수를 직접 옮기거나 교체하고 역할과 팀 지시를 바꾸면 4개 지표가 즉시 반응한다.
- 결과에서 실제 기준 전술과 내 전술의 변화량을 확인하고 다시 설계할 수 있다.

### 3. 완성도 25점

- 회원가입, 결제, API 키, 서버 없이 정적 웹에서 실행된다.
- 원본 데이터 장애가 심사에 영향을 주지 않도록 검증한 소규모 파생값만 번들에 포함한다.
- Playwright로 실제 경기 세 흐름, 볼 흐름 재생, 패스맵 선택, 공간 지배도, 범용 스튜디오 자동 저장, 모바일 가로 넘침을 검증한다.
- GitHub Actions가 타입 검사·빌드·브라우저 테스트 후 Pages를 배포한다.
- 이벤트 보간, 점유 추정, 휴리스틱 비교의 한계를 화면에서 숨기지 않는다.

### 4. 기획/구현 완성도 20점

- 한 줄 콘셉트와 모든 핵심 기능이 `근거 있는 전술 개입`에 연결된다.
- 관측 데이터, 사용자 입력, 규칙 기반 비교값을 서로 다른 시각 언어와 문구로 구분한다.
- 실제 선수 명단과 경기 이벤트는 출처를 적고, 역할·템플릿·코칭 규칙은 서비스용 데이터로 직접 구성한다.
- README, 라이선스 검토, 데이터 재현 스크립트, UX 구조, 시연 대본, 제출 체크리스트가 함께 제공된다.

## 제출 직전 중단 기준

다음 중 하나라도 실패하면 기능을 더 추가하지 않고 안정화한다.

1. 첫 화면부터 결과까지 5분 안에 완료할 수 없다.
2. 모바일에서 가로 스크롤이 생기거나 드래그·버튼이 겹친다.
3. 실제 관측값과 추정·휴리스틱 값의 구분을 설명할 수 없다.
4. 새로고침 후 범용 스튜디오 저장 상태가 사라진다.
5. 배포 URL, GitHub, 영상 중 하나라도 로그아웃 상태에서 열리지 않는다.
